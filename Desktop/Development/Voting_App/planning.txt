Voting App functionality

1. user signIn/ signUp
2. see the list of candidates
3. vote one of the candidates after voting user cant vote again
4. there is a route which shows the list of candidates and their live vote counts sorted by their vote counts
5. user data must contain their one unique ID proof named: Aadhar card number
6. there should be one admin which maintains the table of candidates but should not be able to vote
7. user can change their password
8. user can only login with aadhar card number and password

Routes

User Authentication:
    /signUp: POST - Create a new user account.
    /login: POST - Log in to an existing account. [ Aadhar card number + password ]

Voting:
    /candidates: GET - Get the list of candidates.
    /vote/:candidateId: POST - Vote for a specific candidate.

Vote Counts:
    /vote/counts: GET - Get the list of candidates sorted by their vote counts.

User Profile:
    /profile: GET - Get the user's profile information.
    /profile/password: PUT - Change the user's password.

Admin Candidate Management:
    /candidates: POST - Create a new candidate.
    /candidates/:candidateId: PUT - Update an existing candidate.
    /candidates/:candidateId: DELETE - Delete a candidate from the list.
